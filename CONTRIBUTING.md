# Contributing to VaultixIMQ / Содействие проекту VaultixIMQ

[English](#english) | [Русский](#русский)

---

## English

Thank you for your interest in contributing to VaultixIMQ! We welcome contributions from everyone.

### Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally.
3.  **Install dependencies**:
    *   [Go](https://go.dev/) (v1.22+)
    *   [Node.js](https://nodejs.org/) (v20+)
    *   [Wails CLI](https://wails.io/docs/gettingstarted/installation) (`go install github.com/wailsapp/wails/v2/cmd/wails@latest`)
4.  **Run in development mode**: `wails dev`

### How to Contribute

*   **Reporting Bugs**: Please use GitHub Issues to report bugs. Provide as much detail as possible, including steps to reproduce.
*   **Feature Requests**: Open an issue to discuss new features before starting any work.
*   **Pull Requests**:
    *   Create a new branch for your changes.
    *   Follow the existing code style.
    *   Ensure your changes don't break existing functionality.
    *   Submit a descriptive Pull Request.

### Code of Conduct

Please be respectful and professional in all interactions within this project.

## Русский

Спасибо за ваш интерес к проекту VaultixIMQ! Мы рады любому вкладу в развитие приложения.

### С чего начать

1.  **Форкните репозиторий** на GitHub.
2.  **Клонируйте свой форк** локально.
3.  **Установите зависимости**:
    *   [Go](https://go.dev/) (v1.22+)
    *   [Node.js](https://nodejs.org/) (v20+)
    *   [Wails CLI](https://wails.io/docs/gettingstarted/installation) (`go install github.com/wailsapp/wails/v2/cmd/wails@latest`)
4.  **Запустите в режиме разработки**: `wails dev`

### Как внести вклад

*   **Сообщения об ошибках**: Пожалуйста, используйте GitHub Issues. Опишите проблему максимально подробно, включая шаги для её воспроизведения.
*   **Предложения новых функций**: Откройте новую задачу (Issue) для обсуждения функционала перед началом работы.
*   **Pull Request**:
    *   Создайте новую ветку для ваших изменений.
    *   Придерживайтесь существующего стиля кода.
    *   Убедитесь, что ваши изменения не ломают текущий функционал.
    *   Создайте информативное описание Pull Request.

### Правила поведения

Пожалуйста, будьте вежливы и профессиональны во всех взаимодействиях внутри этого проекта.
